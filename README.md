# PRS Results Repository

This repository stores public PRS/KRS source artifacts generated by PRS Downloader.

## Structure

- `index.json` - global index of companies available in the repository
- `companies/` - one canonical folder per company, keyed by KRS

## Company folder rule

Each company has exactly one canonical folder:

`companies/<krs>-<slug>/`

Example:

`companies/0000002554-parexel-polska/`

Inside that folder:

- `latest/` - current authoritative source set for the company
- `history/` - up to 3 older source sets

## Latest folder

The `latest/` folder is the source of truth for downstream AI analysis.

It contains only:

- `manifest.json`
- `sprawozdanie_finansowe.xml`
- `sprawozdanie_zarzadu.pdf`

## Operational rule

When a new result is published for the same company:

- move the previous `latest` snapshot to `history`
- keep only the 3 newest history snapshots
- write the new authoritative set to `latest`
- use KRS as the primary identity key
