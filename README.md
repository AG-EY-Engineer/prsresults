# PRS Results Repository

This repository stores public PRS/KRS result artifacts generated by PRS Downloader.

## Structure

- `index.json` - global index of companies available in the repository
- `companies/` - one canonical folder per company, keyed by KRS

## Company folder rule

Each company should have exactly one canonical folder:

`companies/<krs>-<slug>/`

Example:

`companies/0000002554-parexel-polska/`

Inside that folder:

- `latest/` - current authoritative result set for the company
- `history/` - optional older snapshots

## Latest folder

The `latest/` folder is the source of truth for downstream AI analysis.

It should contain:

- `summary.md`
- `manifest.json`
- key PDF files
- optional HTML visualization and assets

## Operational rule

When a new result is published for the same company:

- update `latest/`
- do not create a brand new top-level company folder
- use KRS as the primary identity key
